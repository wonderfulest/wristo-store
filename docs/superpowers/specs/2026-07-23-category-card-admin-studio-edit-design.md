# 分类卡片管理员 Studio 编辑入口设计

## 背景

Wristo Store 的所有分类详情页都通过 `Categories.vue` 渲染共享的 `ProductCard.vue`。管理员目前可以在卡片内的管理面板中点击文字版 `Studio` 按钮，但缺少位于应用图片附近、可快速识别的编辑入口。

## 目标

- 在所有 `/categories/:slug` 分类页的应用卡片上增加 edit 小图标。
- 图标仅对管理员显示，普通访客的卡片保持不变。
- 点击图标直接在新窗口打开该应用对应的 Studio 设计。
- 保留卡片主体点击进入 Store 商品详情的现有行为。

## 组件与交互

`Categories.vue` 向 `ProductCard.vue` 传入一个明确的管理员编辑入口开关，使共享卡片在首页、搜索、品牌和排行榜等其他页面不自动出现新图标。

`ProductCard.vue` 仅在以下条件同时满足时渲染图标：

- 页面开启管理员编辑入口；
- 当前用户拥有管理员角色；
- 产品存在非空 `designId`。

入口使用现有 Iconify 图标体系中的铅笔图标，定位在应用图片右上角，并沿用卡片的颜色、圆角、阴影和交互节奏。按钮提供英文 `title` 和 `aria-label`：`Edit in Studio`，保证鼠标提示和辅助技术可识别。

点击按钮时阻止事件冒泡，避免同时进入 Store 商品详情；随后复用 `openStudioDesign(designId)`，在新窗口打开 `https://studio.wristo.io/design?id=<designId>`。现有移动端 Studio 限制提示继续生效。

## 边界与异常处理

- 普通用户不渲染按钮，也不会因样式占位改变卡片布局。
- 缺少 `designId` 的产品不渲染按钮，避免无效跳转。
- 本次不移除管理员面板中的文字版 `Studio` 操作。
- 本次不修改 Studio 路由、登录、权限或产品接口。
- 不影响当前工作区中商品详情图库相关的未提交改动。

## 验收与验证

- 验证分类页已开启管理员编辑入口，其他 `ProductCard` 使用方未开启。
- 验证按钮同时受页面开关、管理员身份和有效 `designId` 控制。
- 验证点击按钮调用现有 Studio 跳转方法并阻止卡片详情跳转。
- 运行聚焦测试、`npm run build` 和 `git diff --check`；无关基线失败单独说明。

## 非目标

- 不在首页、搜索、排行榜或品牌页显示新图标。
- 不向非管理员提供复制或编辑设计能力。
- 不重做卡片视觉或管理员管理面板。

# Store 角色感知计数展示设计

## 目标

在 `wristo-store` 中统一应用数量和下载量的展示规则：只有拥有 `ROLE_ADMIN` 的用户可以看到精确数字；游客、普通用户、商家及其他所有角色只能看到模糊数字。

本规则覆盖所有 Store 可见入口，包括商家列表与详情、分类与系列、应用卡片与详情，以及套餐或订阅中展示的包含应用数量。

## 范围

- 应用数量使用现有 `formatApproxAppCount` 模糊规则。
- 下载量使用现有 `formatApproxDownloadCount` 模糊规则。
- 管理员统一使用 `formatExactCount`。
- 评分、价格、页码、搜索结果分页和其他非应用/下载计数不变。
- 不改变 API 返回结构或数据库数据，只收敛前端展示策略。

## 方案

新增共享的角色感知计数格式化组合函数，由它读取用户角色并提供：

- `formatDisplayAppCount(value)`
- `formatDisplayDownloadCount(value)`

相关组件不再自行重复管理员判断，而是统一调用这两个方法。这样可以保持已有模糊区间不变，同时避免新增页面再次直接输出精确数字。

套餐、订阅等商品规格中的应用数量也遵循相同规则，不保留精确展示例外。

## 数据流

1. 页面或组件获得原始应用数量或下载量。
2. 共享组合函数从用户 Store 读取当前角色。
3. `ROLE_ADMIN` 走精确格式化；其他身份走对应模糊格式化。
4. 组件只渲染格式化后的字符串。

空值、非法值和非正数继续沿用现有工具函数的兜底行为。

## 测试与验收

- 先增加会失败的回归测试，证明仍存在直接输出精确应用数量的入口。
- 覆盖管理员与非管理员两种身份的应用数量和下载量格式化结果。
- 扫描 Store 可见组件，确保应用数量和下载量不再直接渲染原始值。
- 运行相关测试和 `npm run build:app`。
- 在可用的本地页面中验证至少一个管理员和一个非管理员展示入口；若本地身份或数据环境无法提供对应状态，明确记录未完成的浏览器验证范围。

## 非目标

- 不调整模糊区间算法。
- 不隐藏指标本身。
- 不修改后台接口权限或服务端脱敏策略。
- 不改动与本需求无关的现有 SEO 脚本和测试文件。

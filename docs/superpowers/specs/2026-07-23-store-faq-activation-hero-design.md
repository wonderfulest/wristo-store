# Store FAQ 菜单与激活轮播设计

## 背景

Wristo Store 已经提供 FAQ 页面和 `/already-purchased` 激活表单，但 FAQ 菜单入口目前只对拥有 bundle 权益的用户显示，首页轮播也没有专门解释激活权益的卡片。此次调整让所有访客都能发现帮助内容，并从首页直接进入已有激活流程。

## 目标

- 桌面和移动菜单始终展示 FAQ。
- 首页顶部新增一张“如何激活，享受权益”主题轮播卡片。
- 激活卡片的主按钮跳转到当前语言对应的 `/already-purchased`。
- 保留 Studio 入口现有的 bundle 权限限制。
- 不修改激活表单、后端接口、登录或权益判断规则。

## 交互与视觉

### FAQ 菜单

FAQ 沿用现有菜单位置、图标、文案和本地化路径生成逻辑。只移除 FAQ 自身的 bundle 可见性条件；Studio 的可见性条件不变。直接访问 FAQ 的路由行为保持不变。

### 激活轮播

新增独立的 `activation` 轮播项，面向所有访客展示，不参与 bundle 过滤。卡片沿用现有首页 Hero 的排版、圆角、阴影、轮播点、暂停和响应式行为，使用与激活语义一致的图标和视觉元素，避免引入一套新的首页组件。

卡片内容包括：

- 表达激活主题的 eyebrow。
- “如何激活，享受权益”对应的本地化标题。
- 简短说明：输入购买邮箱和手表显示的六位数字激活码即可同步权益。
- 一个明确的主按钮，进入当前语言对应的 `/already-purchased`。
- 专属轮播点无障碍标签。

卡片使用紧凑操作模式，不展示无关的购买或 Studio 次级按钮。视觉区域采用轻量的激活码/权益插画表达；优先通过组件内结构和现有图标完成，不依赖新的远程资源。

## 数据与路由

轮播项继续由 `HomeBanner.vue` 的本地静态配置驱动。新增 `goToActivation` 动作，通过 `addLocaleToPath('/already-purchased', currentLocale)` 生成目标路径，并交给 Vue Router 跳转。现有轮播索引保护逻辑继续覆盖权限变化，不新增后端请求。

新增用户文案写入现有 i18n 消息表。至少保证当前支持的语言均有对应键，避免切换语言后回退成键名；翻译风格跟随同文件现有首页 Hero 文案。

## 边界与异常处理

- 激活轮播对已登录、未登录、有 bundle 和无 bundle 用户均可见。
- FAQ 对所有用户可见，但本次不改变 FAQ 内容和路由访问控制。
- `/already-purchased` 的表单校验、接口错误及成功反馈维持现状。
- 轮播在减少动态可见项或启用 `prefers-reduced-motion` 时继续使用现有安全行为。
- 不改动当前工作区中与商品详情图库相关的未提交文件。

## 验收与验证

- 更新聚焦测试，验证桌面及移动菜单的 FAQ 不再受 bundle 条件控制，而 Studio 仍受控制。
- 验证首页配置包含公开的激活轮播项、正确的本地化路由和激活 CTA。
- 验证新增 i18n 键存在，轮播索引与现有可见项过滤兼容。
- 运行相关 Node 测试和 Store 构建；若仓库存在无关基线失败，单独记录，不混同为本次改动失败。

## 非目标

- 不重新设计 FAQ 页面或激活表单。
- 不新增 FAQ 权限守卫。
- 不改变 `/code` 的用途或跳转关系。
- 不修改 bundle、会员或 Premium 权益语义。
- 不新增后端接口、埋点或 CMS 配置。

# 应用详情页轮播图与管理员就地管理设计

## 背景与目标

Store 应用详情页已经读取管理员上传的 `share` 图片，并通过 `ProductImageGallery` 展示主图与缩略图。现有页面缺少明确的手动轮播控制，管理员也只能前往独立的 Share Image Library 页面维护图片。

本次改造把应用详情页作为图片展示和就地管理入口：普通用户可以手动浏览图片；管理员可以在同一轮播区域添加、删除并调整分享图片顺序。应用原始主图固定为第一张，不允许在此处删除或排序。

## 范围

### 包含

- 详情页主图左右手动切换。
- 主图下方缩略图横向滚动和点击切换。
- 移动端左右滑动切换。
- 键盘方向键切换和现有大图预览。
- 管理员在缩略图行内批量添加图片。
- 管理员删除单张分享图片。
- 管理员通过拖拽调整顺序，移动端提供左移、右移操作。
- 后端提供原子化批量排序接口。

### 不包含

- 自动播放。
- 修改或删除应用原始主图。
- 在详情页编辑图片标题、ALT 文本或其他元数据。
- 取消现有独立 Share Image Library 管理页面。

## 用户体验

### 普通用户

轮播第一张始终是应用原始主图，后续图片按照 `share` 图片的 `sortOrder` 升序排列。图片多于一张时，主图左右两侧出现上一张、下一张按钮，并首尾循环。主图下方展示缩略图条，点击缩略图切换主图，当前缩略图自动滚动到可见区域。手机端支持左右滑动，桌面端支持键盘左右方向键。点击主图继续使用现有的大图预览能力。

普通用户只有一张可用图片时隐藏切换按钮和缩略图条；管理员仍保留缩略图管理行，以便从只有固定主图的初始状态添加图片。加载失败的图片从当前可用图片集合中移除；如果当前图片失败，则切换到相邻可用图片，全部失败时显示现有占位状态。

### 管理员

管理员看到与普通用户相同的轮播，并在缩略图条末尾看到“＋ 添加图片”卡片。选择文件后直接上传，支持 PNG、JPEG、WebP，单张不超过 10 MB，每个应用最多保留 8 张 `share` 图片；固定主图不计入数量限制。上传中禁用重复操作，成功后把新图片追加到现有顺序并保持当前页面位置。

每张 `share` 缩略图显示管理员操作层：删除按钮和拖拽手柄。删除需要二次确认；删除当前图片后选中下一张，删除末张时选中上一张，最后回到固定主图。固定主图不显示这些管理操作。

桌面端通过拖拽缩略图排序，移动端同时提供左移、右移按钮。放下或移动完成后乐观更新界面并提交完整顺序；接口失败时恢复原顺序并显示错误提示。排序范围只包含 `share` 图片，固定主图始终位于首位。

## 组件边界

### `ProductDetail.vue`

负责页面级数据和管理员操作编排：

- 加载公开详情和公开分享图片。
- 管理员登录时加载可管理的分享图片数据。
- 调用上传、删除和排序接口。
- 维护上传、删除、排序状态。
- 操作成功后用接口返回列表更新本地状态，不整页刷新。

### `ProductImageGallery.vue`

保持为无 API 依赖的展示与交互组件：

- 合并固定主图和分享图片，固定主图始终排第一。
- 管理手动切换、缩略图可见性、触摸滑动、键盘和大图预览。
- 接收管理员编辑模式和忙碌状态。
- 通过事件上报添加文件、删除图片和新顺序，不直接调用后端。

这种边界确保普通用户的画廊展示不依赖管理员接口，管理失败也不会破坏公开图片状态。

## 数据与接口

现有 `product_images.sort_order` 字段继续作为分享图片的排序来源。现有上传接口追加图片并分配递增的 `sortOrder`，现有删除接口继续删除单张关系和对应资源。

新增管理员批量排序接口：

```text
PUT /api/admin/products/{appId}/share-images/order
Content-Type: application/json

{
  "productImageIds": [31, 18, 42]
}
```

接口在单个事务中完成以下校验和更新：

1. 应用存在。
2. ID 列表无空值、无重复值。
3. 列表与该应用当前所有有效 `share` 图片完全一致。
4. 每条关系都属于当前应用且类型为 `share`。
5. 按数组位置把 `sortOrder` 规范化为从 `0` 开始的连续整数。

成功后返回按新顺序排列的完整管理员分享图片列表。采用完整 ID 列表而不是逐条更新，可避免并发或部分失败造成重复排序值和中间状态。

公开接口继续只返回可公开字段，并按 `sortOrder ASC, createdAt ASC` 排序。固定主图由详情数据提供，不写入 `share` 排序接口。

## 状态与错误处理

- 上传前在前端校验格式、大小和剩余数量，后端继续执行相同约束作为最终保障。
- 上传成功使用返回的完整列表，避免前端猜测服务端 ID 和排序值。
- 删除成功后从本地列表移除目标；失败时保持列表不变。
- 排序采用快照回滚：提交前保留旧顺序，失败时恢复。
- 管理操作进行时禁用冲突操作，防止同一图片同时删除和排序。
- 图片加载失败只影响当前展示集合，不删除服务端数据。
- 所有管理员接口仍由现有管理员鉴权保护，普通用户不渲染管理控件。

## 可访问性与响应式

- 上一张、下一张、添加、删除、左移、右移均提供明确的英文 `aria-label`，与 Store 面向用户的英文界面保持一致。
- 当前缩略图使用 `aria-current`。
- 轮播获得焦点时响应左右方向键。
- 删除确认框可通过键盘完成。
- 触摸滑动设置最小距离阈值，避免普通垂直滚动被误判。
- 遵循 `prefers-reduced-motion`，关闭非必要的平滑滚动和位移动画。

## 测试与验收

### Store 前端

- 单元/契约测试覆盖固定主图与分享图片合并顺序、首尾循环、图片失败回退和删除后的选中项。
- 组件契约测试覆盖管理员控件仅在编辑模式出现，以及添加、删除、排序事件载荷。
- API 契约测试覆盖批量排序请求方法、路径和请求体。
- 执行 `npm run build`。

### API 后端

- Orchestrator 测试覆盖成功排序、跨应用 ID、非 `share` 类型、重复 ID、缺失 ID 和空列表场景。
- Controller 测试覆盖管理员权限、请求绑定和响应结构。
- Mapper/Service 测试覆盖连续 `sortOrder` 更新与查询顺序。
- 执行相关 Maven 测试；条件允许时执行 `mvn test`。

### 手工验收

- 普通用户只能看到手动轮播，不能看到管理控件。
- 管理员可在详情页添加、删除、拖拽排序，刷新后顺序保持。
- 移动端可以滑动浏览并通过左移、右移调整顺序。
- 固定主图始终第一张且无法删除、移动。
- 只有一张图片、达到 8 张上限、上传失败、删除失败和排序失败时界面状态正确。
